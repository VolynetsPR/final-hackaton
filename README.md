В данном руководстве подробно шаг за шагом разобраны базовые темы, позволяющие создавать веб-приложения на Python с использованием Flask, сохранять данные и работать с базами данных форматов SQLite. Руководство написано максимально простым языком специально для новичков.

## 1. Создание веб-приложений с использованием микрофреймворка Flask

**Flask** — это небольшая программа-движок (фреймворк) для языка Python, с помощью которой можно быстро запустить настоящий веб-сайт. Она берёт на себя всю сложную работу по принятию запросов от браузера и ответу на них.

**Первые шаги.** Чтобы начать, создайте файл (например, `app.py`) и напишите:
```python
from flask import Flask

# 1. Мы создаем экземпляр приложения. 
# __name__ — это специальная переменная Python, которая передает фреймворку имя текущего файла.
app = Flask(__name__) 

# 2. Декоратор @app.route. Он связывает адрес в браузере с функцией в Python.
# '/' означает главную страницу сайта (например, http://127.0.0.1:5000/)
@app.route('/') 
def hello_world():
    # То, что эта функция возвращает (return), пользователь увидит на экране
    return 'Привет, мир!'

# 3. Запуск сервера.
if __name__ == '__main__':
    # Метод run() запускает локальный сервер.
    # Параметр debug=True очень важен для новичков: 
    # он автоматически перезагружает сервер при изменении кода и показывает ошибки прямо в браузере.
    app.run(debug=True)
```

## 2. Создание и использование HTML-шаблонов (render_template)

Писать весь дизайн сайта прямо внутри функций Python в виде текста (`return '<h1>Привет</h1>...'`) — очень неудобно. Flask умеет «рендерить» (то есть прочитывать и отдавать пользователю) отдельные HTML-файлы. 

**Важное правило:** Flask всегда автоматически ищет HTML-документы в папке с названием `templates/` рядом с вашим `app.py`. Если назвать папку иначе, Flask ничего не найдет.

**Структура файлов:**
```text
твой_проект/
  ├─ app.py
  └─ templates/
     └─ index.html
```

**app.py:**
```python
# Нужно импортировать функцию render_template
from flask import Flask, render_template

app = Flask(__name__)

@app.route('/')
def index():
    # Эта функция возьмет файл templates/index.html, подготовит его и отправит в браузер.
    return render_template('index.html')
```

## 3. Передача параметров в шаблон

Если вы просто отдаете `index.html`, страница будет статичной. Но мы можем передавать туда переменные из Python. Шаблонизатор (программа, которая обрабатывает шаблон), встроенный во Flask, называется **Jinja2**.

Вы можете передавать любые типы данных: текст (строки), числа, списки и словари (dict).

**Python:**
```python
@app.route('/profile')
def profile():
    # Допустим, мы выгрузили данные пользователя из базы
    user_data = {
        'name': 'Иван',
        'age': 25,
        'city': 'Москва'
    }
    
    # Чтобы не писать render_template('profile.html', name=user_data['name'], age=user_data['age']...)
    # можно использовать **user_data
    # Две звездочки (**) распаковывают словарь в именованные параметры, 
    # так что шаблон получит переменные name, age и city по отдельности.
    return render_template('profile.html', **user_data)
```

**HTML (templates/profile.html):**
Чтобы вывести переменную в HTML, нужно написать ее название в двойных фигурных скобках `{{ название_переменной }}`.

```html
<h1>Профиль пользователя</h1>
<!-- Туда подставится 'Иван' -->
<p>Имя: {{ name }}</p>
<!-- Туда подставится 25 -->
<p>Возраст: {{ age }}</p>
```

## 4. Условия и циклы в HTML-шаблонах

Иногда нам нужно скрыть кнопку, если пользователь не вошел в систему, или вывести таблицу со списком из 100 товаров. Использовать обычный Python прямо в HTML нельзя, но Jinja2 предоставляет свою похожую грамматику (в фигурных скобках со знаком процента `{% ... %}`).

**Условия (`if` / `elif` / `else`):**
Не забудьте, что в шаблонах все условия нужно закрывать, написав `{% endif %}`.

```html
{% if age >= 18 %}
    <p>Вам разрешен доступ к полному каталогу фильмов.</p>
{% elif age >= 14 %}
    <p>Доступны только фильмы рейтинга PG-13.</p>
{% else %}
    <p>Доступны только мультики.</p>
{% endif %}
```

**Циклы (`for`):**
Позволяют пробежаться по списку. Тоже обязательно закрываются с помощью `{% endfor %}`.

```html
<!-- Допустим, мы передали из Python список покупок: shopping_list = ['Хлеб', 'Молоко', 'Творог'] -->
<h2>Список:</h2>
<ul>
{% for item in shopping_list %}
    <!-- Цикл создаст тег <li> для каждого элемента -->
    <li>{{ item }}</li>
{% endfor %}
</ul>
```

## 5. Использование функции url_for для статического контента

Создавать веб-сайт без картинок и стилей скучно. Такие файлы называются "статикой". По правилам Flask их нужно класть **строго** в папку с названием `static/`.

```text
твой_проект/
  └─ static/
     ├─ css/
     │  └─ style.css
     └─ images/
        └─ logo.png
```

Использовать прямые ссылки вида `<img src="/static/images/logo.png">` считается плохим тоном: если вы перенесете сайт на другой сервер или измените настройки, пути сломаются. Поэтому нужно использовать встроенную функцию `url_for()`.

**Пример (templates/base.html):**
```html
<head>
    <!-- url_for('static', filename='путь_к_файлу_внутри_папки_static') -->
    <link rel="stylesheet" href="{{ url_for('static', filename='css/style.css') }}">
</head>
<body>
    <img src="{{ url_for('static', filename='images/logo.png') }}" width="150" alt="Логотип">
</body>
```

## 6. Передача параметров в адресной строке (Динамические URL)

Вам не нужно писать отдельную функцию для каждого пользователя `/user/ivan`, `/user/misha` и т.д. Можно сделать путь динамическим — с параметром (переменной).

Чтобы сказать Flask, что часть пути изменяемая, оберните ее в треугольные скобки `<...>`. Эту же переменную нужно добавить в параметры функции.

```python
# 1. Параметры в пути URL (Path variables)
# Если человек перейдет на /user/admin, в переменную username попадет строка 'admin'
@app.route('/user/<username>')
def show_user_profile(username):
    return f'Добро пожаловать в профиль игрока: {username}'

# Можно принудительно указать тип данных параметра. 
# <int:id> означает, что Flask будет реагировать только на целые числа.
@app.route('/post/<int:post_id>')
def show_post(post_id):
    # post_id автоматически станет числом (int) в Python, а не строкой.
    return f'Отображение статьи с номером {post_id}'
```

**Параметры с вопросительным знаком (Query String):**
Иногда параметры передаются в конце ссылки после знака вопроса, например: `http://сайт.ру/search?word=flask`.
Чтобы получить это значение, нужно импортировать объект `request` из Flask.

```python
from flask import request

@app.route('/search')
def search():
    # request.args хранит все, что идет после знака вопроса.
    # метод get('word') заберет значение 'flask'.
    # Вторым параметром ('') мы задаем значение по умолчанию, если ничего не передали.
    query = request.args.get('word', '') 
    return f'Вы искали в Яндекс: {query}'
```

## 7. Обработка HTML-форм как способ взаимодействия с пользователем

Формы — это поля ввода на сайте (текстовые поля, галочки, кнопки), через которые пользователь отправляет вам информацию (регистрация, логин, отправка комментария).

У форм есть два основных метода отправки:
- **`GET`**: данные добавляются прямо в адресную строку (как в поиске выше). Небезопасно для паролей.
- **`POST`**: данные передаются "внутри" запроса. Для любых форм с паролями и личными данными нужно строго использовать метод `POST`.

**1. HTML-сторона (templates/login.html):**
```html
<!-- method="POST" указывает скрытый тип передачи  -->
<!-- action="/login" указывает, на какой URL отправить данные  -->
<form method="POST" action="/login">
    <!-- Атрибут 'name' невероятно важен. Именно по нему Python поймет, какое это поле -->
    <input type="text" name="user_login" placeholder="Введите логин">
    <input type="password" name="user_password" placeholder="Введите пароль">
    
    <button type="submit">Отправить</button>
</form>
```

**2. Python-сторона (app.py):**
```python
from flask import request, render_template

# В декораторе ОБЯЗАТЕЛЬНО нужно явно разрешить методы GET (чтобы показать страницу) 
# и POST (чтобы принять данные из формы).
@app.route('/login', methods=['GET', 'POST'])
def login():
    # Если пользователь нажал кнопку 'Отправить' (тип запроса POST)
    if request.method == 'POST':
        # Для метода POST мы достаем данные из request.form, а не request.args
        # Используем те самые названия, которые писали в атрибут name="..." в HTML!
        login = request.form.get('user_login')
        pwd = request.form.get('user_password')
        
        # Тут мы сверяем пароль с базой данных...
        return f'Вы ввели логин: {login} и пароль: {pwd}'
    
    # Если человек просто зашел на страницу (метод GET), показываем ему HTML с формой
    return render_template('login.html')
```

## 8. Набор инструментов Bootstrap (подключение стилей)

Вместо того чтобы писать сотни файлов CSS-кода для того, чтобы кнопки были круглыми, красивыми и меняли цвет при наведении, можно использовать **Bootstrap**. Это огромная библиотека уже готовых стилей, написанная профессионалами. 

Что дает Bootstrap:
- Удобную "сетку", чтобы сайт сам сжимался под экраны телефонов.
- Готовые теги для кнопок, меню, карточек товаров.

Легче всего подключить его прямо по ссылке из интернета (CDN).

**Пример шаблона (base.html):**
```html
<!DOCTYPE html>
<html>
<head>
    <!-- Вставляем эту ссылку в <head> вашего сайта -->
    <link href="https://cdn.jsdelivr.net/npm/bootstrap@5.3.0/dist/css/bootstrap.min.css" rel="stylesheet">
</head>
<body>
    <!-- Класс 'container' центрирует контент. 'mt-4' делает отступ (margin-top) 4 единицы -->
    <div class="container mt-4"> 
        
        <!-- text-center ставит текст по центру, text-primary красит его в синий цвет -->
        <h1 class="text-center text-primary">Мой сайт</h1>
        
        <!-- Для колонок (сеток) нужно создать класс 'row' (строка) -->
        <div class="row mt-3"> 
            
            <!-- col-md-6 говорит: займи 6 колонок из 12 возможных (то есть ровно половину ширины) -->
            <div class="col-md-6">
                <!-- btn — делает элемент кнопкой. btn-success красит ее в зеленый. w-100 делает ее широкой на 100% -->
                <button class="btn btn-success w-100">Войти</button>
            </div>
            
            <div class="col-md-6">
                <!-- btn-danger красит в красный (отмена, удаление) -->
                <button class="btn btn-danger w-100">Выйти</button>
            </div>
            
        </div>
    </div>
</body>
</html>
```

## 9. Работа с CSV файлами (чтение и запись)

CSV (Comma-Separated Values) — это текстовый файл, в котором данные разделены запятыми (или точкой с запятой). Он очень прост и открывается как в Python, так и в программах вроде Microsoft Excel (в виде обычной таблицы).

**Пример содержимого (data.csv):**
```csv
Имя;Возраст;Город
Иван;20;Москва
Анна;22;Казань
```

**Чтение файла в Python:**
```python
import csv

# Открываем файл. 'r' значит read (чтение). encoding='utf-8' нужен для поддержки русского языка.
with open('data.csv', mode='r', encoding='utf-8') as file:
    # delimiter=';' сообщает программе, что колонки разделены точкой с запятой
    reader = csv.DictReader(file, delimiter=';')
    
    # Проходимся по каждой строке таблицы
    for row in reader:
        # Теперь row — это словарь. Можно обращаться к колонке по ее названию!
        print(row['Имя'], "живет в", row['Город'])
```

**Запись в файл:**
```python
import csv

# Допустим, мы собрали эти данные из формы регистрации
new_users = [
    {"Name": "Игорь", "Age": 28},
    {"Name": "Ева", "Age": 19}
]

# 'w' значит write (запись). newline='' убирает лишние пустые строки между записями в Windows.
with open('output.csv', mode='w', encoding='utf-8', newline='') as file:
    # Указываем, какие колонки будут в нашей таблице.
    writer = csv.DictWriter(file, fieldnames=["Name", "Age"], delimiter=';')
    
    # Сначала записываем "шапку" таблицы с названиями колонок
    writer.writeheader()
    # Затем разом записываем весь список наших словарей
    writer.writerows(new_users)
```

## 10. Работа с JSON файлами (чтение и запись)

JSON — это повсеместно используемый формат для сохранения данных в виде текста. Визуально он один-в-один похож на словари (`dict`) и списки (`list`) в Python. Он удобнее CSV, так как позволяет хранить "вложенные" данные (например, у пользователя может быть внутри список его покупок).

**Чтение JSON (превращение файла в Python-словарь):**
```python
import json

# r - чтение
with open('settings.json', 'r', encoding='utf-8') as file:
    # Метод load загружает текст из файла и превращает его в словарь Python
    data = json.load(file) 
    print(data['theme']) 
```

**Запись данных в JSON:**
```python
import json

user = {
    "login": "admin",
    "roles": ["read", "write"]
}

# w - запись (файл будет создан, если его нет, или перезаписан)
with open('user.json', 'w', encoding='utf-8') as file:
    # Метод dump берет Python-словарь 'user' и записывает в 'file'
    # Параметры для новичка:
    # indent=4 : Форматирует текст как код с переносами строк и табуляцией в 4 пробела (чтобы было читаемо для человека)
    # ensure_ascii=False : Важный параметр! Без него русские слова запишутся как \u0430\u0431.
    json.dump(user, file, indent=4, ensure_ascii=False)
```

## 11. Использование библиотеки matplotlib для графиков

Эта библиотека позволяет рисовать наглядные графики. В веб-приложениях мы генерируем график в Python, сохраняем его как `.png` картинку в папку `static/` и затем показываем пользователю на HTML-странице через `url_for()`.

```python
import matplotlib.pyplot as plt

# Допустим, мы собрали эти данные из БД
месяцы = ["Янв", "Фев", "Мар", "Апр"]
продажи = [1500, 2300, 1800, 3100]

# Дополнительные настройки:
# Создаем "холст" графика (фигуру) заданного размера (в дюймах), 8 в ширину, 5 в высоту
plt.figure(figsize=(8, 5))

# Строим линию (plot)
# Параметры: color - цвет, marker - отображать точки на узлах ('o' — кружки), linestyle - тип линии ('--' пунктир)
plt.plot(месяцы, продажи, color='blue', marker='o', linestyle='--', label='Выручка')

plt.title('График продаж по месяцам') # Заголовок графика
plt.xlabel('Месяц')                   # Название горизонтальной оси
plt.ylabel('Сумма (в рублях)')        # Название вертикальной оси
plt.grid(True)                        # Включаем сетку, чтобы было удобнее смотреть цифры
plt.legend()                          # Выводим подсказку (легенду), которая возьмет текст из параметра label

# Сохраняем готовую картинку в папку static/
# Параметр dpi=300 сделает картинку очень четкой (высокое разрешение), чтобы она не была мыльной.
plt.savefig('static/image/plot.png', dpi=300)

# Обязательно очищаем холст за собой. 
# Если этого не сделать и нарисовать второй график, он наложится прямо поверх первого!
plt.close()
```

# Очистка холста, если в одном скрипте строим несколько графиков
plt.close()
```

## 12. Создание и работа с базами данных в SQLiteStudio

**SQLiteStudio** — это десктопная программа с графическим интерфейсом для работы с базами данных SQLite (`.db` или `.sqlite`).
Поскольку SQLite представляет собой всего один файл на диске, не требуется устанавливать сервер БД (как MySQL или PostgreSQL).

**Как работать:**
1. Открыть SQLiteStudio.
2. `Database -> Add a database`. Указать путь к файлу базы данных вашего проекта (например, `instance/app.db`).
3. В интерфейсе можно создавать таблицы (ПКМ -> `Create a table`), настраивать столбцы (Primary Key, Data Types — `INTEGER`, `TEXT`, `REAL`).
4. Вкладка `Data` позволяет просматривать, добавлять или удалять строки вручную.
5. Вкладка `Query` используется для написания SQL запросов вручную для тестирования.

## 13. Написание запросов к БД (SQL). Запросы SELECT

SQL (Structured Query Language) позволяет извлекать нужные данные.

**Базовый синтаксис SELECT:**
```sql
-- Выбор всех столбцов из таблицы users
SELECT * FROM users;

-- Выбор конкретных столбцов
SELECT username, email FROM users;
```

**Фильтрация данных (WHERE):**
```sql
-- Выбор всех, чье имя 'Иван'
SELECT * FROM users WHERE name = 'Иван';

-- Использование операторов >, <, >=, <=, != (или <>)
SELECT * FROM products WHERE price > 1000;

-- Несколько условий (AND / OR)
SELECT * FROM users WHERE age >= 18 AND status = 'active';

-- Поиск по подстроке (LIKE) '%an%' найдет Ivan, Anna и т.д.
SELECT * FROM users WHERE email LIKE '%@gmail.com';
```

**Сортировка и лимиты:**
```sql
-- Сортировка по возрасту (по возрастанию - ASC, по убыванию - DESC)
SELECT * FROM users ORDER BY age DESC;

-- Получение только первых 5 записей (удобно для пагинации)
SELECT * FROM users LIMIT 5;

-- Получение с пропуском первых 10 записей (OFFSET)
SELECT * FROM users LIMIT 10 OFFSET 10;
```
