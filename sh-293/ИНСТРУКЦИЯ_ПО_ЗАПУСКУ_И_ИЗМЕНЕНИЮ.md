# Подробная инструкция по запуску и изменению проекта

Проект — веб-приложение для анализа результатов заключительного этапа ВсОШ по информатике.

Стек проекта:

- Python
- Flask
- HTML-шаблоны Jinja2
- CSS
- SQLite
- matplotlib для графиков
- BeautifulSoup для парсинга HTML-файлов результатов

---

## 2. Что нужно установить перед запуском

Нужен установленный Python версии 3.10 или выше.

Проверить версию Python можно командой:

```bash
python3 --version
```

или, если используется Windows:

```bash
python --version
```

Если Python не установлен, его нужно скачать с официального сайта:

```text
https://www.python.org/downloads/
```

При установке на Windows желательно поставить галочку:

```text
Add Python to PATH
```

## 3. Установка зависимостей

В корневой папке проекта выполните команду:

```bash
python3 -m pip install -r requirements.txt
```

На Windows команда может выглядеть так:

```bash
python -m pip install -r requirements.txt
```

Файл `requirements.txt` содержит зависимости:

```text
Flask
beautifulsoup4
matplotlib
```

Если установка прошла успешно, можно переходить к созданию базы данных.

---

## 4. Создание базы данных

Перед первым запуском сайта нужно создать SQLite-базу данных из приложенных HTML и CSV файлов.

Для этого выполните:

```bash
python3 init_db.py
```

На Windows:

```bash
python init_db.py
```

После успешного выполнения появится файл:

```text
data/olympiad.sqlite3
```

В консоли должно появиться сообщение примерно такого вида:

```text
База данных создана: /.../data/olympiad.sqlite3
```

Скрипт `init_db.py` делает следующее:

1. удаляет старую базу, если она есть;
2. создаёт новую SQLite-БД;
3. создаёт таблицы;
4. парсит все HTML-файлы из папки `uploads/`;
5. загружает результаты участников;
6. загружает школы Москвы и Санкт-Петербурга из CSV;
7. связывает участников с их школами;
8. копирует `standings2.css` в папку `static/`.

## 7. Проверочный сценарий по требованиям кейса

В кейсе указано, что приложение должно работать после удаления исходных HTML и CSV, если БД уже создана.

Проверить это можно так:

### Шаг 1. Остановите сайт

Если Flask запущен в терминале, нажмите:

```text
Ctrl + C
```

### Шаг 2. Удалите старую БД

Удалите файл:

```text
data/olympiad.sqlite3
```

### Шаг 3. Создайте БД заново

```bash
python3 init_db.py
```

### Шаг 4. Временно уберите исходные файлы

Можно переименовать папку `uploads`, например:

```bash
mv uploads uploads_backup
```

На Windows можно просто переименовать папку вручную.

### Шаг 5. Запустите сайт

```bash
python3 app.py
```

Если сайт открывается и работает, значит данные действительно берутся из SQLite-БД, а не напрямую из HTML/CSV.

После проверки можно вернуть папку обратно:

```bash
mv uploads_backup uploads
```

```python
DIPLOMA_INVITED = 383
WINNERS_LIMIT = int(DIPLOMA_INVITED * 0.08)
```

## 12. Как добавить новую страницу

Допустим, нужно добавить страницу `/about`.

### Шаг 1. Создайте шаблон

Создайте файл:


```python
@app.route('/about')
```

В файле `templates/base.html` добавьте:

```html
<a class="nav-link" href="/about">О проекте</a>
```

### Шаг 4. Перезапустите Flask

Остановите сервер через `Ctrl + C` и запустите снова:

```bash
python3 app.py
```

## 13. Как изменить фильтры таблицы результатов

Фильтры отображаются в:

```text
templates/results_form.html
```

А обрабатываются в:

```text
app.py
```

конкретно в функции:

```python
results_table()
```

Фактическая фильтрация находится в:

```text
services.py
```

функция:

```python
filtered_result_rows(...)
```

Например, если нужно добавить фильтр только по участникам с баллом больше 300:

1. добавить поле в форму `results_form.html`;
2. считать параметр в `results_table()`;
3. передать его в `filtered_result_rows()`;
4. внутри `filtered_result_rows()` отфильтровать строки.

Список классов задан в `app.py`:

```python
CLASS_FILTERS = {

    '10_down':'10 класс и младше',
```

Если нужно добавить, например, только 8 класс:

```python

```

После этого нужно добавить обработку в `services.py` в функции `filtered_result_rows()`:

```python
if class_filter == '8' and cls != 8:
    continue
```

## 15. Как изменить правила выбора таблицы по времени

Логика выбора таблицы находится в `services.py`:

```python
select_snapshot(con, tour_choice, time_mode='final', minute=None)
```

Сейчас работает так:

- если выбран итоговый результат — берётся последняя таблица выбранного тура;
- если выбран момент времени — берётся самая свежая таблица, созданная не позже указанной минуты.

Если таблицы до указанного времени нет, сайт показывает:

```text
Результаты ещё не сформированы
```

## 16. Как изменить графики

Графики создаются в `app.py`.

Основные функции:

```python
chart_points(pid, kind, mode)
chart(pid, kind, mode)
```

`chart_points()` собирает точки графика.

`chart()` рисует график через matplotlib.

Чтобы изменить цвет линии, найдите:

```python
ax.plot(x, y, marker='o', linewidth=2.2, color='#2b6cb0')
```

и замените `color`.

Например:

```python
color='#dc2626'
```

Чтобы изменить размер графика, найдите:

```python
fig, ax = plt.subplots(figsize=(8, 3.8), dpi=130)
```

и поменяйте `figsize` или `dpi`.

Если файла `data/olympiad.sqlite3` нет, выполните:

```bash
python3 init_db.py
```

---

### Ошибка: порт 5000 занят

Если Flask пишет, что порт занят, можно запустить на другом порту.

В конце `app.py` замените:

```python
app.run(debug=True)
```

на:

```python
app.run(debug=True, port=5001)
```

После этого сайт будет доступен по адресу:

```text
http://127.0.0.1:5001/
```

### После изменения HTML/CSS ничего не поменялось

Попробуйте:

1. обновить страницу через `Ctrl + F5`;
2. перезапустить Flask;
3. проверить, что меняется правильный файл в папке `templates/` или `static/`.

---

### После изменения Python-кода сайт работает по-старому

Остановите сервер:

```text
Ctrl + C
```

и запустите снова:

```bash
python3 app.py
```

## 19. Рекомендуемый порядок внесения изменений

1. Сделать копию проекта или нужного файла.
2. Изменить один небольшой участок.
3. Перезапустить Flask.
4. Проверить страницу в браузере.
5. Если изменение связано с БД — выполнить `python3 init_db.py`.
6. Проверить, что остальные страницы не сломались.

## 21. Что лучше не менять без необходимости

Не рекомендуется без необходимости менять:

- структуру таблиц в `db.py`, если не нужно добавлять новые данные;
- шаблон `standings.html`, если не понимаете, как Jinja2 подставляет данные;
- парсер `parser_utils.py`, если формат исходных HTML не изменился;
- функцию `get_school()`, если сопоставление школ работает достаточно хорошо.

Но внешний вид безопасно менять в:

```text
static/app.css
static/bootstrap.min.css
```

А тексты страниц — в:

```text
templates/index.html
templates/results_form.html
templates/stats_regions.html
templates/stats_schools.html
templates/participant.html
```

## 22. Краткое описание архитектуры

Приложение работает по такой схеме:

```text
HTML/CSV файлы → init_db.py → SQLite БД → app.py → templates → браузер
```

То есть сайт не читает HTML/CSV напрямую при обычной работе.

Сначала данные загружаются в SQLite, затем Flask берёт всё из базы данных.

Это соответствует требованию кейса: после создания БД исходные HTML/CSV файлы можно удалить, и приложение продолжит работать.
